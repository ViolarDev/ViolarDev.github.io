Обзор

Система сообщений предоставляет реализацию широковещательных оповещений. Передача данных от "вещателя" (broadcaster) к "подписчикам" (subscribers) осуществляется через экземпляры класса MessagesHub. "Подписка" осуществляется через метод Subscribe, отправка сообщения через [Broadcast]()Vdev.Messaging.MessagesHub.Broadcast). В один момент времени в системе может быть сколько угодно экземпляров MessagesHub - ои будут работать в полной изоляции: нет возможности чтобы подписчики одного hub'а получили сообщение отправленное через другой.

Подписка

На данный момент реализовано два мехнизма подписки на события:

  • ручной: через непосредственное указание обработчика сообщения через вызов статического метода Subscribe конректного типа сообщения;
  • автоматический: если метод класса должен быть обработчиком сообщения, то он должен быть помечен аттрибутом MessageHandlerAttribute и последующей [автоматической подпиской]((Vdev.Messaging.MessagesUtils.AutoSubscribe).

Ручная подписка.

using Vdev.Messaging;
// int - parameter of the message.
class MyMessage : Message<MyMessage, int>
{}
// ...
void Initialize()
{
// subscribe
MyMessage.Subscribe((value)
{ //value of int
});
}

Автоматическая подписка.

using Vdev.Messaging;
// int - parameter of the message.
class MyMessage : Message<MyMessage, int>
{}
//...
class MyClass : IDisposable
{
SubscriptionInfo m_subscription;
[MessageHandler(typeof(MyMessage))]
void HandleMessage(int value)
{
// ... handle message.
}
void Initialize()
{
m_subscription = MessageUtils.AutoSubscribe(this);
}
public void Dispose()
{
if (m_subscription != null)
{
m_subscription.Dispose();
m_subscription = null;
}
}
}
