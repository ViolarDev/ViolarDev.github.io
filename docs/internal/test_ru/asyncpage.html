Обзор 

Асинхронные операции предлагают альтернативу Unity coroutine.

Отличия Vdev.Async и Unity coroutines:

  • возврат результата;
  • возврат ошибок (через механизм исключений);
  • Tracing операций: позволяет отследить "путь" операции: точки асинхронных вызовов (кто/где создаёт, где используется);
  • комбинирование/композиция операций ;
  • синхронизация операций исполняющихся в разных потоках;
  • "переиспользование" операций : возможность вызывать "await" для уже завершённой операции (кэширование операции, а не самого результата);
  • возможность одновременного использования одной операции в нескольких точках синзронизации (ожидать завершения операции могут одновременно несколько "дочерних" операций);

IAsyncOp<R>

Асинхронная операция представленна классом Vdev.Async.IAsyncOp<R "IAsyncOp<R>". В качестве generic параметра тип возвращаемого операцией значения. Если операция не возвращает значения, то должен быть использован Nothing.
